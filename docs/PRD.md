# Tourlinks Calculator PRD

## 1. Product Overview
- 제품명: Tourlinks Calculator
- 제품 유형: 골프 회원권 손익 시뮬레이터
- 목적: 골프 회원권 상품 기획 시 회사 손익성과 고객 체감가치를 빠르게 비교 검토한다.
- 핵심 가치: 연도별 가정과 지역별 운영 구조를 반영해 `BEP`, `LTV`, 고객 절감 효과를 즉시 확인한다.

## 2. Target Users
- 골프 회원권 상품기획 담당자
- 사업개발 및 제휴 담당자
- 운영 및 재무 검토 담당자

## 3. Core User Jobs
- 신규 회원권 구조의 손익성을 1차 검토한다.
- 지역 조합과 가격 정책 변화가 손익에 미치는 영향을 확인한다.
- 고객에게 매력적인 상품인지 고객 절감 효과 기준으로 판단한다.
- 여러 시나리오를 비교해 우선안과 대안을 정리한다.

## 4. Product Scope
### Included
- 시나리오 생성, 수정, 삭제, 이름 변경
- 공통 조건 입력
- 지역별 조건 입력
- 연도별 이용률 입력
- 연도별 해지율 입력
- 연도별 일괄 해지 환불금 입력
- 회사 LTV, 회사 BEP 계산
- 고객 절감액, 고객 순이익, 고객 BEP 계산
- 민감도 분석
- 시나리오 간 비교
- JSON Import/Export

### Excluded
- 회원 관리, 계약, 결제 기능
- 외부 시스템 연동
- 실제 정산 처리
- 권한 관리

## 5. Data Model
### Scenario Common Fields
- 시나리오명
- 분양가(`salePrice`)
- 연회비(`annualFee`)
- 연 고정 손익(`annualFixedProfit`)
- 계약기간(`contractYears`)
- 연차별 해지율(`churnByYear`)
- 연차별 일괄 해지 환불금(`refundAmountByYear`)

### Region Fields
- 지역명
- 연간 라운드 한도(`roundLimitByRegion`)
- 구장 매입가(`courseCostByRegion`)
- 회원 판매가(`memberPriceByRegion`)
- 동반 판매가(`companionPriceByRegion`)
- 비회원 판매가(`publicPriceByRegion`)
- 라운드당 평균 회원 수(`membersByRegion`)
- 라운드당 평균 동반 수(`companionsByRegion`)
- 연차별 이용률(`usageByRegion`)

## 6. Key Calculations
### Company View
- 라운드당 회원 손익 = `(회원 판매가 - 구장 매입가) x 회원 수`
- 라운드당 동반 손익 = `(동반 판매가 - 구장 매입가) x 동반 수`
- 라운드당 총손익 = `회원 손익 + 동반 손익`
- 지역 연간 라운딩 손익 = `라운드당 총손익 x 실제 라운딩 횟수`
- 연도별 손익 = `가입비 반영액 + 생존 고객 기준 연회비 + 연간 라운딩 손익 - 환불비용`
- 기대 고정 손익 = `가입비 반영액 + 생존 고객 기준 연회비 - 환불비용`
- LTV = `계약기간 전체 누적 회사 손익`

### Customer View
- 회원 절감액 = `(비회원 판매가 - 회원 판매가) x 회원 수`
- 동반 절감액 = `(비회원 판매가 - 동반 판매가) x 동반 수`
- 라운드당 총절감액 = `회원 절감액 + 동반 절감액`
- 고객 순이익 = `총 절감액 누적 - 분양가`
- 고객 BEP = `고객 누적 절감액이 분양가를 회수하는 시점`

### Retention View
- 연도별 잔존율은 해지율을 반영해 감소한다.
- 환불비용은 해지 고객 수와 일괄 해지 환불금을 기준으로 계산한다.
- 연도별 이용률은 지역별로 별도 입력 가능하다.

## 7. Screens
### 1) Input
- 공통 조건 입력
- 지역 추가/삭제 및 지역별 조건 입력
- 지역별 연도 이용률 수정
- 연도별 해지율 및 환불금 수정
- 통합 집계와 가중평균 표시

### 2) BEP Analysis
- 회사 LTV와 고객 순편익을 연평균 라운드 기준으로 시각화
- 회사 손익분기와 고객 투자 회수 구간 확인
- 지역별 마진/절감 구조 확인

### 3) Dashboard
- 계약기간 전체 LTV 확인
- 연도별 잔존율, 이용률, 환불비용, 누적 LTV 확인
- 손익 구조 요약

### 4) Sensitivity
- 동반 수 변화에 따른 LTV
- 회원 수 변화에 따른 LTV
- 이용률 변화에 따른 LTV

### 5) Compare
- 복수 시나리오의 LTV, 연간 평균 손익, 회사 BEP, 고객 BEP 비교
- 시나리오별 누적 LTV 추이 비교

## 8. Functional Requirements
- 사용자는 최대 4개의 시나리오를 관리할 수 있어야 한다.
- 사용자는 시나리오당 최대 6개의 지역을 입력할 수 있어야 한다.
- 계약기간이 바뀌면 이용률, 해지율, 환불금 배열이 자동 보정되어야 한다.
- 모든 계산 결과는 입력 변경 즉시 갱신되어야 한다.
- 사용자는 현재 시나리오 집합을 JSON으로 내보내고 다시 불러올 수 있어야 한다.

## 9. Non-Functional Requirements
- 입력 후 계산 결과는 지연 없이 즉시 반영되어야 한다.
- 테이블과 차트는 데스크톱 기준으로 주요 정보를 빠르게 확인할 수 있어야 한다.
- 비재무 담당자도 이해 가능한 용어를 사용해야 한다.

## 10. Success Metrics
- 회원권 초안 검토 시간이 기존 대비 크게 단축된다.
- 두 개 이상의 시나리오를 빠르게 비교해 기획 의사결정 속도가 빨라진다.
- 회사 손익성과 고객 가치를 동시에 검토하는 공통 도구로 사용된다.
